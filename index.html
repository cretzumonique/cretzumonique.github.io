Professional Self-Assessment

Completing the Bachelor of Science in Computer Science program at Southern New Hampshire University has been a transformative experience, particularly because I began the program in 2023 with no prior formal background in computer science. My professional foundation as a Quality Engineer in the prototyping and manufacturing industry shaped how I approached every course, every project, and ultimately this capstone. Quality engineering is fundamentally about defect prevention, root cause analysis, process documentation, and continuous improvement — and I have found that these are not merely transferable soft skills, but the same underlying discipline that distinguishes professional software development from amateur coding.

Collaborating in a Team Environment

Throughout the program, collaboration was woven into coursework that simulated real development environments. In CS 250 (Software Development Lifecycle), team-based sprint planning and stand-up style check-ins mirrored Agile practices, and in CS 320 (Software Testing, Automation, and Quality Assurance) peer code reviews required giving and receiving constructive technical feedback — a skill directly exercised again in this capstone's Milestone One code review. In my professional role, I regularly facilitate cross-functional meetings between engineering, production, and quality teams. Each milestone in this capstone explicitly responded to instructor feedback from the prior milestone, which is itself a form of asynchronous collaboration. When Milestone Two received feedback to expand security controls and testing, that feedback directly shaped the priorities for Milestone Four.

Communicating with Stakeholders

Professional-quality communication was a thread running through every milestone. The code review video required translating technical code analysis into a walkthrough appropriate for peers or a manager. Each milestone narrative required explaining why an artifact was selected, what specific skills were demonstrated, and how course outcomes were achieved — in language precise enough for a technical evaluator but clear enough to serve as a writing sample for a future employer. In my current role, I write quality reports and corrective action documentation that must satisfy both engineering teams who need technical detail and management who need risk and impact summarized at a glance. The ePortfolio itself is the culmination of this skill: presenting four milestones of technical work as a single, navigable, professional narrative.

Data Structures and Algorithms

The Algorithms and Data Structures enhancement in Milestone Three was the most technically concentrated demonstration of growth in this area. Building the Trend Analysis Engine required implementing five distinct algorithms — a sliding window for the 7-day rolling average achieving O(n) time complexity, binary search for date-range lookup at O(log n), TimSort-based multi-criteria sorting at O(n log n), trend direction detection, and a single-pass streak calculator. The most significant realization was that algorithmic efficiency is not an academic exercise — the difference between an O(n) sliding window and an O(n*k) naive recalculation is invisible with 10 data points but becomes significant with 365 daily entries, which is a realistic usage pattern for the application.

Software Engineering and Database

Software engineering competency was demonstrated in Milestone Two through a substantial architectural refactor: progress dashboard, dirty flag pattern, DiffUtil adapter, and comprehensive input validation and Javadoc. That milestone received positive instructor feedback noting strong software engineering growth. Database competency was demonstrated in Milestone Four: Write-Ahead Logging mode, foreign key enforcement, CHECK constraints at the schema level, and data-preserving ALTER TABLE migration strategy. Both enhancements adopted industry-standard practices specifically because they improve the application beyond minimum functional requirements — demonstrating the kind of engineering judgment expected in professional development.

Security

Security was the area of most significant personal growth. The original Weight Tracker artifact stored user passwords as plaintext — a critical vulnerability (CWE-256) I did not recognize in CS 360. By the time of this capstone, after the broader program emphasis on a security mindset, identifying and mitigating that exact vulnerability in my own earlier work was a powerful demonstration of growth. Milestone Four implemented salted SHA-256 password hashing with a transparent migration path, centralized SQL injection screening, and a SecurityAudit class documenting seven identified vulnerabilities by CWE identifier, severity, and mitigation status.

How the Artifacts Fit Together

All three enhancements were applied to a single artifact — the Weight Tracker Android Application from CS 360 — to demonstrate the realistic lifecycle of a software project. Enhancement One established a more maintainable architecture. Enhancement Two built on that architecture to add the Trend Analysis Engine. Enhancement Three hardened the data layer underlying both prior enhancements. Together they tell a coherent story: a functional but unrefined mobile application was transformed, through a structured and feedback-driven process, into a more architecturally sound, algorithmically capable, and secure application — directly mirroring the incremental improvement work expected of a software engineer or technical project manager in industry.

Professional Direction

This capstone has confirmed my professional direction toward technical project management and quality assurance leadership in software development. The combination of my quality engineering background with the technical competencies demonstrated in this ePortfolio positions me to bridge the gap between engineering teams and organizational stakeholders. I plan to pursue PMP certification as a complement to this degree, and this ePortfolio will serve as the technical foundation supporting that transition.

Code Review Highlights

Software Engineering: No architectural pattern (MVVM absent), business logic embedded in Activity classes, no input validation, orphaned SMS method.

Algorithms & Data Structures: Entries ordered by insertion ID rather than date, no sorting or filtering, no trend analysis or rolling average, O(n) full scan on every load with no optimization.

Databases: Critical plaintext password storage, inconsistent query safety (rawQuery with string concatenation), destructive onUpgrade strategy, no DAO abstraction layer.
